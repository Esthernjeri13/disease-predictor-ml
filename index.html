How MediPredict Works
MediPredict uses a Gaussian Naive Bayes classifier trained on a labelled symptom-disease dataset. The model applies Bayes' theorem to compute the posterior probability of each disease given the observed symptoms, then returns the disease with the highest probability.
  • User selects present symptoms as binary inputs (1 = present, 0 = absent)
  • Gaussian NB computes log-likelihood for each disease class using learned mean and variance per feature
  • Log-likelihoods are summed with class priors to get posterior scores
  • Softmax converts log-posteriors to probabilities for display
  • Top prediction + confidence ranking are shown to the user
📐
The Math Behind It
Gaussian Naive Bayes computes:

P(C|X) ∝ P(C) × ∏ P(xᵢ|C)

Where P(xᵢ|C) follows a Gaussian distribution:

P(x|C) = exp(−(x−μ)²/2σ²) / √(2πσ²)

In practice, log-space arithmetic is used to avoid underflow. The model learns one μ and σ² per feature per class from the training data. At inference time, the model runs entirely in the browser — no server required.
🗃️
Dataset Details
The training dataset was synthetically generated based on clinically-informed symptom profiles for 8 diseases.
  • 1,600 patient records — 200 per disease class
  • 20 binary symptoms — fever, cough, rash, jaundice and more
  • Symptom probabilities derived from real clinical literature profiles
  • 5% random noise applied to simulate real-world variation
  • Stratified 80/20 train-test split for unbiased evaluation
⚠️
Limitations & Ethics
  • This model is trained on synthetic data and is not validated on real clinical records
  • Symptom overlap between diseases (e.g. Malaria vs Influenza) reduces accuracy in ambiguous cases
  • Binary symptom encoding ignores symptom severity and duration
  • The tool is for educational purposes only — never use it to replace professional medical advice
  • Real clinical deployment would require IRB approval, clinical validation, and regulatory compliance
